A predictive model for task run-time resource usage

  1. 1 Describe your sample
  2. 2 Describe the procedures that were used to collect the data
  3. 3 Describe your variables

1 Describe your sample

  • a:Describe the study population

I want to build a predictive model for task run-time CPU resource usage. This model uses statistical regression models to get the estimated CPU resource allocation for the next run epoch.

  • b:Report the level of analysis studied

It’s a group analysis studied.

  • c:Report the number of observations in the data set.

It’s only one observations in the data set. It’s the CPU usage.

  • d:Describe your data analytic sample

The sample is the task run-time CPU usage in any epoch. We collect every epochs of a task’s run-time as the sample.

2 Describe the procedures that were used to collect the data

  • a:Report the study design that generated that data

We use the experiment to generate those datas. We use one second as a epoch. In other words, we collect one data per second.

  • b:Describe the original purpose of the data collection

We want to know the CPU usage in every epochs of a task’s run-time and hope that we can build a model to predict the CPU usage in next epoch base the some epoch ago.

  • c:Describe how the data were collected

We use the supervisory program to collect the data.

  • d:Report when the data were collected

We collected the data when the task is run.

  • e: Report where the data were collected

The data is save in the data.txt file with the pairs, and the key is the NO. ,the value is the task run-time CPU resource usage in this epoch.

3 Describe your variables

  • a:Describe what your explanatory and response variables measured

The explanatory variables are ten CPU usages in ten task run-time epochs and the response variable is the CPU usage in next task run-time epoch.

  • b:Describe the response scales for your explanatory and response variables.

    ??

  • c:Describe how you managed your explanatory and response variables.

    We use ten CPU usages in ten task run-time epochs as a explanatory variable and the next epoch CPU usage as the response variable.
